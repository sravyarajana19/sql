Frontend
----------
1. The part of a website that the user interacts with directly is called frontend.
2. In frontend it is also referred as client side of the application.
3. It includes everything that user experience directly.
4. HTML, CSS, JS, BootStrap are the languages used for frontend development .

Backend
-------
1. Backend is the server side of the website.
2.It stores and arrange the data also make sure everything on the client side of the web site works fine. 
3. In backend there are some languages like java ,python , PHP,.net ....etc.

API
---
It means application programming interface, which is the collection of communication protocols used by various programs to communicate b/w them.

storage areas
-----------
As a part of our applications we required to store data like customers information or billing information or false information.....etc.
To store this data we required storage areas.

There are 2 types of storage areas such as:

1. Temporary : there are the memory areas where the data will be stored temporally.
example --> all JVM memory areas(like Heap Area, Stack Area, Method Area, PC Registers, Native Method Stack).
once JVM Shutdown all these memory areas will cleared automatically .

2. Permanent: also known as persistent storage areas. Here we can stored data permanently
example--> Files System , Databases, Data Warehouse , Bigdata...etc.

File Management System :
------------------------
def:  A file management system is a system where data is stored in files on the operation system , each application program must handle its own storage data, retrieval and updating .
file system can be provided by the local operating system .
file Systems are best suitable for to store very less amount of information (like KB).

example--> a library management system storing books in separate text files like, books.txt, members.txt

Limitations of file management system 
--------------------------------------
1. Data Redundancy : same data is stored in multiple files.
2. Data Inconsistency : update in one file may not be reflected in another.
3. poor data security : no proper accessed control.
4. Difficult Data Retrieval : searching requires custom programs.
5. Integrity issues: no constrains ( ex:- roll no. uniqueness)
6. scalability issues : hard to manage as data grows.

To overcome the above problem of file system , we should go for databases.
===================================================================================================================================================================

Data Base Management System 
------------------------
 def : a DBMS is a collection of program that enables users to create, manage, and manipulate databases.
 it acts as an interface between user and data.

adv :- 
1. we can store huge amount of information in databases.
2.query language support is available for every data base and hence they can perform database operations very easily .
3.To access data presenting database compulsory user name  and password must be required hence data is secured .
4.Inside database data will be stored in the form of tables while developing database table schemas ,database admin follow various normalization techniques and we can implement various constrains like unique key constrains , primary key constrains .... etc. which prevent data duplication, hence there is no chance of data inconsistency problems

limitations
------------
1.Data base cannot stored huge amount of information like tera bytes(1000 gb data)
2.Data base can provide support only for structured data .(tabular data or relational data) and cannot provide support for semi structured data like (xml files) and unstructured data (video file, audio files, images...etc.)

to overcome these problems we should go for advanced storage areas like bigdata , data warehouse....etc.
